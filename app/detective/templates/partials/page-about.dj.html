About Detective.io

What is Detective.io?

Detective.io is a platform to store and mine all the data you have on a precise topic. You first design the ontology, then upload it to Detective.io, which creates an input interface and a front-end automatically. The front-end lets you display all the information and search it using the powerful graph search.

What the heck is an ontology?

It's a way to describe a topic. Think of it as all the sentences you can think of that link two entities in a certain area. Let's take the example of the ontology that describes a fast food restaurant:

  • An employee produces burgers.
  • An employee works in the kitchen.
  • A customer buys burgers.
All these statements have the same structure: a subject (employee, customer), a predicate (produces, works, buys) and an object (burgers, the kitchen). Predicates represent relationships and subjects and objects are called classes. Each relationship or class can then have properties (an employee can have a contract type and a salary, for instance). The ontology is the sum of the classes, the relationships and their properties.

So, it's like Freebase?

No. Detective.io is a graph database and makes use of concepts from the semantic web. But Detective.io is very different from Freebase in a key way: It focuses on specific investigations. If you mention the "British Government" in an investigation on development projects in Equatorial Africa, you don't mean the same thing as "British Government" in an investigation on London's politicians. It's impossible to represent the whole knowledge in a single database, were it only because concepts don't have the same meaning in different situations.

Is this the semantic web of linked data?

In a way, yes. Detective.io structures all the data you give it and uses OWL as an ontology language. But Detective.io's ontologies are not (yet) fully compatible with other services, such as DBpedia.

Why did you build it?

We were working on an investigation about innovative energy projects in developing countries thanks to a grant from the European Journalism Center. We couldn't find a database that would let us store all our data, so we built one and made it reusable.

Who is behind Detective.io?

Journalism++ SAS. We are a company specialized in data-driven journalism and are fully independent.

Can I use it myself?

Detective.io has no point-and-click interface to create new investigations yet. You can download the source code and install Detective.io on your own server. You can also contact us if you need a helping hand.
